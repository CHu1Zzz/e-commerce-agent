## 1. 产品概述

### 1.1 背景

电商客服面临大量重复性咨询（订单查询、物流追踪、退换货政策等），人工处理效率低且成本高。通过 AI Agent 自动处理高频问题，可显著降低人力成本并提升响应速度。

### 1.2 产品目标

构建一个基于 LangChain 的电商客服 Agent MVP，能够：

- 🤖 自动回答常见电商咨询问题
- 📦 查询订单状态与物流信息
- 🔄 处理退换货申请流程
- 👤 识别无法处理的问题并转交人工客服

### 1.3 MVP 范围界定

| 做 ✅ | 不做 ❌ |
|------|--------|
| 文字对话交互 | 语音/视频客服 |
| 订单查询、物流追踪 | 实际下单/支付 |
| 退换货申请提交 | 退款审批与执行 |
| 商品信息问答（LLM 直接回答） | RAG 知识库检索 |
| 简单 FAQ 应答 | 个性化推荐 |
| 人工转交 | 多轮复杂工单系统 |

## 2. 目标用户

| 角色 | 场景 | 需求 |
|------|------|------|
| **C 端消费者** | 购物后遇到问题 | 快速获取订单/物流/退换货信息 |
| **客服人员** | 处理消费者咨询 | 减少重复劳动，专注复杂问题 |
| **运营人员** | 监控客服质量 | 查看 Agent 对话数据与指标 |

## 3. 核心功能

### 3.1 智能对话

- 接收用户自然语言输入
- 识别用户意图（订单查询 / 退换货 / 商品咨询 / 闲聊 / 其他）
- 生成自然、友好的回复
- 维护多轮对话上下文（Session 级 Memory）

### 3.2 订单查询

- **输入**：订单号 或 用户描述（如"我前天买的东西到哪了"）
- **输出**：订单状态、物流轨迹、预计送达时间
- **数据源**：模拟订单 API（MVP 阶段用 Mock 数据）

### 3.3 物流追踪

- 根据订单号查询物流状态
- 以时间线格式展示物流节点
- 预计送达时间提示

### 3.4 退换货申请

- 引导用户填写退换货原因
- 校验是否在退换货时效内
- 生成退换货工单（MVP 阶段写入本地文件/数据库）
- 返回退货地址和注意事项

### 3.5 商品咨询

- 回答常见商品相关问题（尺码建议、退换货政策、常见使用问题等）
- MVP 阶段由 LLM 基于 Prompt 内置知识直接回答，不接入 RAG
- 商品 FAQ 写入 Prompt 的 Few-shot 示例中

### 3.6 人工转交

- 当 Agent 连续 2 次无法理解用户意图时
- 当用户明确要求"转人工"时
- 当涉及退款金额争议时
- 输出转交提示，包含对话摘要

## 4. 用户流程

### 4.1 主对话流程

```
用户发送消息
    │
    ▼
Agent 意图识别
    ├── 订单查询 ──→ 调用订单查询 Tool ──→ 返回订单信息
    ├── 物流追踪 ──→ 调用物流查询 Tool ──→ 返回物流轨迹
    ├── 退换货   ──→ 引导填写信息 ──→ 提交申请 ──→ 返回确认
    ├── 商品咨询 ──→ LLM 直接回答（Prompt 内置 FAQ）
    ├── 闲聊/FAQ ──→ LLM 直接回答
    ├── 转人工   ──→ 生成对话摘要 ──→ 转交提示
    └── 无法识别 ──→ 澄清追问（连续 2 次则转人工）
```

### 4.2 退换货子流程

```
用户：我想退货
    │
    ▼
Agent：请问您的订单号是？
    │
    ▼
用户：ORD-20260530-001
    │
    ▼
Agent 查询订单 → 校验时效
    ├── 超出时效 → 告知无法退换
    └── 在时效内 → 询问退换货原因
                      │
                      ▼
              用户选择原因（质量问题/不想要了/尺码不对/其他）
                      │
                      ▼
              生成退换货工单 → 返回退货地址 + 注意事项
```

### 5.2 LangChain 核心组件

| 组件 | 选型 | 说明 |
|------|------|------|
| **LLM** | MiniMax (langchain-minimax) | 国产模型，性价比高，中文能力强 |
| **Agent 类型** | ReAct Agent | 推理+行动，适合多 Tool 场景 |
| **Memory** | ConversationBufferWindowMemory | 保留最近 K 轮对话 |
| **Tool 框架** | @tool 装饰器 | 定义 Agent 可调用的工具 |
| **商品咨询** | LLM + Prompt 内置 FAQ | MVP 不接入 RAG，FAQ 写入 Prompt |
| **Output Parser** | StrOutputParser | 解析 LLM 输出 |

## 7. Prompt 设计

### 7.1 System Prompt（核心）

```
你是"小帮"，一位友好、专业的电商客服助手。

## 你的职责
- 回答用户关于订单、物流、退换货、商品的咨询
- 使用可用的工具查询真实数据，不要编造信息
- 当无法确定答案时，诚实告知并建议转人工

## 常见商品 FAQ
- 尺码建议：T恤偏大一码，建议选小一码；裤子按正常尺码选购
- 退换货政策：签收后 7 天内可退换，商品需保持原包装
- 发货时间：下单后 24 小时内发货，节假日顺延
- 支付方式：支持微信、支付宝、银行卡

## 行为规范
- 语气友好但专业，使用口语化表达
- 回答简洁，重点突出
- 涉及金额、时效等信息必须基于工具返回的真实数据
- 退换货申请必须走完整流程，不能跳步
- 连续 2 次无法理解用户意图时，主动建议转人工

## 当前时间
{current_time}
```

## 8. 验收标准

| # | 验收项 | 标准 |
|---|--------|------|
| 1 | 订单查询 | 输入订单号能正确返回订单状态与商品信息 |
| 2 | 物流追踪 | 能展示物流时间线与预计送达时间 |
| 3 | 退换货申请 | 能引导完成退换货流程并生成工单 |
| 4 | 商品咨询 | 能基于 Prompt 内置 FAQ 回答常见商品问题 |
| 5 | 多轮对话 | 能在 5 轮内保持上下文连贯 |
| 6 | 人工转交 | 触发条件正确，对话摘要完整 |
| 7 | 幻觉控制 | 不编造订单/物流/商品信息，未查到则如实告知 |
| 8 | 响应时间 | 单次回复 < 5s（不含 LLM 首 token 延迟） |

## 11. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM 幻觉导致错误信息 | 高 | Prompt 约束 + Tool 优先策略 + 关键信息校验 |
| 意图识别不准确 | 中 | 充足的 Few-shot 示例 + 兜底转人工机制 |
| MiniMax API 稳定性 | 中 | 重试机制 + 降级提示 |
| API 成本超预期 | 低 | MiniMax 性价比高 + 缓存高频问题 |
